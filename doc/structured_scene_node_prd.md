# PRD: 专业化分镜节点重构 (Structured Scene Node)

## 1. 产品背景
目前的分镜节点仅能展示纯文本，且 Markdown 语法无效。为了复刻 OmniScript 的专业体验，分镜节点需要具备解析剧本结构、渲染富文本以及展示多维情感信息的能力。

---

## 2. 核心功能需求

### 2.1 增强型 Markdown 渲染
- **要求：** 分镜节点必须支持标准 Markdown（加粗、斜体、列表、引用）。
- **脚本增强：** 自动识别 `角色：对话` 模式。
  - 角色名自动加粗并使用不同颜色。
  - 对话内容使用标准剧本字体。
  - 括号内的情绪（如 `(生气)`）自动渲染为浅色并附加对应 Emoji。

### 2.2 结构化数据展示 (OmniScript 风格)
- **多行剧本流：** 不再是单一文本块，而是由多条 `ScriptLine` 构成的流式布局。
- **环境描述：** 以 `>` 开头的行渲染为灰色斜体背景，表示环境说明。
- **高光标注：** 标记为“高光”的行在节点上拥有发光边框或特殊背景色。

### 2.3 交互式编辑
- **所见即所得：** 编辑模式下，支持实时预览 Markdown 效果。
- **智能辅助：** 在编辑区下方提供“常用角色”和“情绪 Emoji”快捷按钮。

---

## 3. 视觉与 UI 设计规范
- **卡片样式：** 保持 Polaroid 风格，但增加细微的内阴影和纸张纹理。
- **行间距：** 1.6 倍行高，确保剧本易读性。
- **溢出处理：** 支持在节点高度不足时显示滚动条，或提供“查看全文”弹出层。

---

## 4. 技术验收标准
- [ ] 所有的 Markdown 标记在非编辑模式下均能正确转化为对应的 HTML 样式。
- [ ] 性能：在画布上有 20+ 分镜节点时，交互不卡顿。
- [ ] 兼容性：旧的 `content` 纯文本字段应能自动向下兼容。
