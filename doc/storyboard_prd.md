# PRD: 结构化分镜系统 (Structured Storyboard)

## 1. 文档概览
本方案旨在复刻 OmniScript 的核心结构化叙事能力，将其集成至 Canvas 的分镜管理流程中，提升从剧本到分镜制作的专业度和自动化程度。

---

## 2. 核心价值与目标
- **结构化表达：** 将模糊的剧情描述转变为包含角色、台词、情绪、动作的结构化数据。
- **视觉化预览：** 快速识别分镜的情绪基调和关键动作点。
- **创作协同：** 为 AI 视频生成提供标准的 Prompt 原始数据输入。

---

## 3. 用户故事 (User Stories)
1. **角色管理：** 作为创作者，我希望在分镜中明确标注对话者，以便管理复杂的群戏场次。
2. **情绪标注：** 作为导演，我希望用 Emoji 标注每个台词的情绪，直观感受剧情节奏。
3. **时序对齐：** 作为剪辑师，我希望看到每个动作的时间点，以便进行分镜拆解。
4. **快速编辑：** 作为编剧，我希望能在详情面板中像填表一样快速录入台词和动作。

---

## 4. 功能拆解 (Feature Breakdown)

### 4.1 数据模型升级
- **分镜行 (Scene Line)：** 最小数据单元，包含：
  - **角色 (Role)：** 下拉选择或自定义。
  - **内容 (Content)：** 对话或动作。
  - **情绪 (Emotion)：** 预设标签（开心、愤怒等）+ Emoji。
  - **时间戳 (Timestamp)：** 相对场次开始的时间。
- **场次摘要 (Summary)：** 对本场分镜内容的 AI 摘要或人工概述。

### 4.2 交互界面 (UI/UX)
- **多维详情面板 (Detail Panel)：**
  - **剧本页签 (Script Tab)：** 仿 OmniScript 的剧本流展示，左侧角色情绪，右侧内容。
  - **分析页签 (Analysis Tab)：** 展示该场次的关键元素（如：地点、天气、摄影机角度建议）。
- **智能快捷输入：**
  - 在输入台词时，支持用 `(情绪)` 或 `[动作]` 的简写方式快速标注。

### 4.3 剧本联动 (Sync)
- 自动从关联的 `ScriptElement` 解析出上述结构化信息。
- 修改分镜节点内容时，可选同步回原始剧本（反向同步）。

---

## 5. 验收标准
- [ ] 分镜详情面板支持展示结构化的剧本行。
- [ ] 能够通过 UI 添加、删除、修改单行剧本的内容。
- [ ] 分镜卡片在网格视图下能显示情绪摘要（如前三个情绪的 Emoji）。
- [ ] 保持与现有 `ScriptNode` 的兼容性。
