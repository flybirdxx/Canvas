# 项目架构重构建议

## 目标

本建议用于指导 `Canvas` 项目的维护型架构拆解。目标不是重写项目，也不是新增或删除功能，而是在保持现有行为稳定的前提下，逐步降低模块耦合，让后续节点类型、AI 能力、画布交互、执行流程和展示层目录可以更低风险地迭代。

## 本轮保留原则

1. `services/*` 根文件继续保留兼容 re-export，减少调用方一次性 import churn。
2. `useCanvasStore` 的 persistence、migration、history、undo/redo 行为不做语义改动。
3. presentation 迁移只移动展示层文件和修正 import，不新增或删除功能。
4. `.superpowers/` 是本地工具目录，不进入提交范围。

## 验证清单

每轮结构迁移后必须运行：

```bash
npm run lint:tsc
npm run test
git diff --check
```

涉及画布交互时，还需要补充浏览器 smoke，至少覆盖应用能加载、无启动 console error、基础画布交互不报错。
